# Phase 1 Week 2 결산 보고서

## 1. 이번 주 목표 당성
Week 2 의 메인 목표였던 **`통합 인증 시스템 및 보안 체계 구축`** 작업을 100% 온전하게 달성하였습니다.

✅ **인증 시스템 완성도: 100%**

- **Database 스키마 & 규칙 마련**: Firestore 초기화 스크립트 작성 및 Collections Rule 구축 (RBAC 적용) 완료.
- **Firebase Auth 및 트리거 이식**: 신규 가입 감지(`onUserCreate`), Custom Role 부여(`setUserRole`) 구축 완료.
- **소셜 로그인 다중 지원**: 이메일을 포함한 `Kakao` 및 `Naver` 각각 독립된 Oauth 흐름을 Common Interface 에 묶어 병합 처리 완료. 
- **프론트엔드 라우트 블로킹**: 비승인 리퀘스트 차단 모델 구현, 그리고 Custom Claims를 이용해 프론트엔드 레벨(`ProtectedRoute`)에서도 완벽하게 우회 접속 및 권한 외 접근 차단 완료.

## 2. 보안 통제 점검 구조
- API 키들은 프론트와 별개로, 클라우드 환경 내 `.env` 파일과 에뮬레이트 `mock` 프로파일로 엄격히 관리됩니다.
- 소셜 로그인은 Random 발급 `state`를 기반으로 `sessionStorage` 헥사키 체크를 구동, CSRF 공격을 자체 검열하도록 체계화되었습니다.
- Firestore 규칙 위에서, DB 읽기와 쓰기 역시 `request.auth` 의 커스텀 토큰과 Role 분기문을 거치게 되므로 다중으로 보호되고 있습니다.
