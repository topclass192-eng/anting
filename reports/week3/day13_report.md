# Phase 1 Week 3 Day 13 - Campaign Register Step 1 Implementation Report

## 개요
브랜드 사용자가 캠페인을 등록할 때 사용하게 될 **멀티스텝 폼의 1단계(기본 정보)** 및 **임시저장 기능**을 구현했습니다.

## 주요 구현 사항

### 1. 멀티스텝 폼 구조 및 Step 1 화면 (`CampaignRegister.tsx`)
- 상단에 진행 상황을 시각적으로 나타내는 스텝바 (Step 1/3, 2/3, 3/3)를 구성하고, `currentStep` 상태를 기반으로 단계별 렌더링을 구현했습니다.
- **기본 정보 폼 구성**:
  - 캠페인명 (텍스트, 최대 50자)
  - 연결 제품 선택 (`getProducts` Cloud Function과 연동하여 소유 중인 제품 목록 표시)
  - 모집 인원 (1~100명 사이 숫자만 입력 가능)
  - 지역 조건 (체크박스를 이용한 다중 선택 허용)
  - 진행 플랫폼 (인스타그램, 블로그, 틱톡 체크박스 제공)
  - 마감일 (최소 선택 가능 일자를 **오늘로부터 7일 뒤**로 제약)
- 폼 전환 전 유효성 검사 적용(필수값 여부 등 점검).

### 2. 자동 임시저장 로직 (Draft)
- 프론트엔드에서 `setInterval`을 사용해 30초마다 `saveCampaignDraft` 함수를 자동 호출하여 현재까지 작성한 폼 데이터를 임시저장합니다. (완전한 빈 폼일 경우 저장을 생략하여 불필요한 Draft 생성을 방지합니다.)
- 커스텀 **Toast UI**를 구현하여, 30초 자동 저장 시 우하단에 "임시저장되었습니다."라는 메시지를 3초간 표시하고 사라지도록 처리했습니다. (외부 라이브러리에 의존하지 않음)
- 마운트 시 `getCampaignDraft` 함수를 호출하여 이전에 작성 중인 데이터(`status: 'draft'`)가 존재할 경우, "이전에 작성 중인 캠페인이 있습니다. 이어서 작성하시겠습니까?" 팝업을 띄우고 사용자의 선택에 따라 상태값을 복구합니다.

### 3. 클라우드 함수 (`saveCampaignDraft.ts`, `getCampaignDraft.ts`)
- **보안 및 인증**: 요청한 사용자가 브랜드(`role: 'brand'`)인지 검증합니다.
- **`saveCampaignDraft`**: 전달된 데이터를 바탕으로 `campaigns` 컬렉션에 `status: 'draft'`를 마킹하여 업서트(생성 또는 업데이트)합니다.
- **`getCampaignDraft`**: `brandId`와 일치하는 가장 최근의 draft를 1건 조회하여 반환합니다.
