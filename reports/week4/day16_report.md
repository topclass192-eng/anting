# Phase 1 Week 4 Day 16 - Influencer Profile Implementation Report

## 개요
Phase 1의 4주차 첫 번째 개발 과제로, **인플루언서 사용자를 위한 프로필 등록 화면(`Profile.tsx`)**을 구현했습니다. 이를 통해 인플루언서가 가입 후 필수적인 정보 및 SNS 채널 연동 정보를 입력하고, 식별 가능한 고유 코드를 발급받아 활동을 시작할 수 있게 되었습니다.

## 주요 구현 사항

### 1. 인플루언서 기본 정보 폼 구현
- **닉네임 중복 확인 로직**: 최대 20자의 닉네임을 입력받으며, Firestore의 `influencers` 컬렉션을 클라이언트 단에서 쿼리(`where('nickname', '==', value)`)하여 중복 여부를 실시간으로 체크하도록 구현했습니다. 중복 확인을 통과해야만 제출이 가능합니다.
- **자기소개 및 지역**: 최대 100자의 자기소개란과 활동 지역을 선택할 수 있는 드롭다운 메뉴를 구성했습니다.
- **관심 카테고리 (멀티셀렉트)**: '뷰티', '식품', '패션' 등 제공된 카테고리 목록에서 사용자가 자유롭게 다중 선택(최소 1개 이상)할 수 있도록 칩(Chip) 형태의 토글 버튼 UI를 개발했습니다.
- **프로필 이미지 업로드**: 기존 브랜드 개발 단계에서 만들어둔 재사용 가능한 `ImageUpload` 컴포넌트를 활용하여, `influencers/{uid}/profile` 경로에 안전하게 이미지가 업로드되도록 연결했습니다.

### 2. 동적 SNS 채널 등록 UI
- **채널별 토글 (Instagram, Blog, TikTok)**: 인플루언서가 자신이 주로 활동하는 매체를 켜고 끌 수 있는 토글 스위치 형태의 UI를 적용했습니다.
- **상세 입력 활성화**: 각 채널 토글이 활성화(`enabled: true`)될 때만, 해당 채널의 **프로필 URL**과 **팔로워/이웃 수(숫자)**를 입력받는 폼이 나타납니다.
- **최소 1개 제약 조건**: 폼 제출 시 하나 이상의 SNS 채널이 켜져 있고 입력값이 유효한지 검사하는 클라이언트 사이드 Validation을 포함했습니다.

### 3. Seller Code 자동 발급 및 Firestore 저장
- **`sellerCode` 생성**: 클라이언트 단독으로 `ANT-` 문자열과 랜덤한 영숫자 8자리 조합(예: `ANT-B7X9K2P1`)을 즉석 생성하여 문서 저장 시점에 포함시킵니다.
- **데이터베이스 연동**: 폼 제출 시 사용자의 `authUid`를 문서 ID로 하여 `influencers` 컬렉션에 전체 프로필 데이터(SNS 구조체 포함)를 `setDoc`으로 안전하게 저장합니다. 완료 후 인플루언서 메인 대시보드(`/influencer/dashboard`)로 이동합니다.
