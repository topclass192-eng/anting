# Phase 1 Week 4 Day 20 - 인플루언서 대시보드 및 네비게이션 적용

## 개요
Week 4의 마지막 작업으로 인플루언서 사용자가 로그인 후 진입하는 **메인 대시보드(`Dashboard.tsx`)**를 구현하고, 사용자의 권한(역할)에 따라 **하단 네비게이션(`BottomNav.tsx`)**이 동적으로 변경되도록 기능을 확장했습니다.

## 주요 구현 사항

### 1. 하단 네비게이션 동적 라우팅 (`BottomNav.tsx`)
- 기존 브랜드 중심의 고정 탭(홈, 캠페인, 쇼핑, 리포트, 마이)에서 사용자 URL 경로(`location.pathname.startsWith('/influencer')`)에 따라 인플루언서 탭으로 전환되는 로직을 추가했습니다.
- **인플루언서 탭**: 홈, 캠페인, AI콘텐츠(준비중), 쇼핑, 마이.
- `lucide-react` 아이콘을 도입하여 기존 이모지보다 일관성 있고 깔끔한 벡터 아이콘 UI로 개선했습니다. 모바일 하단 안전 영역(`env(safe-area-inset-bottom)`) 대응 패딩을 추가해 최신 모바일 기기에서의 사용성도 확보했습니다.

### 2. 인플루언서 대시보드 UI (`Dashboard.tsx`)
- **프로필 요약 및 수익 카드**:
  - `influencers` 컬렉션에서 사용자 데이터를 가져와 상단 헤더에 닉네임과 등급을 렌더링합니다.
  - "이번 달 페이백"과 "출금 가능 포인트"를 블랙 & 화이트의 고급스러운 카드 디자인(글래스모피즘 라이트 효과 추가)으로 구현했습니다.
  - 고유 **판매자 코드(`sellerCode`)**를 표시하며, 클릭 시 클립보드에 복사되고 '복사됨' 피드백을 주는 유틸리티 버튼을 추가했습니다.
- **진행 중인 캠페인 섹션**:
  - 선발완료 → 배송중 → 수령 → 작성 → 제출의 5단계 진행 프로세스 상태바를 위한 UI 뼈대를 설계했습니다. (향후 applications 데이터 연동 시 카드 형태로 매핑될 예정입니다).
  - 현재 지원 내역이 없을 경우, 부드러운 유도 메시지와 함께 "캠페인 둘러보기" 버튼을 제공하여 탐색 화면으로 이어지게 처리했습니다.
- **AI 추천 캠페인 섹션**:
  - `getCampaigns` 로직을 응용하여 현재 모집 중(`active`)인 최신 캠페인 2개를 쿼리해 렌더링합니다.
  - 각 카드는 썸네일, 브랜드명, 캠페인명 정보를 포함하며 클릭 시 상세 화면으로 바로 이동하도록 `Link`를 연결했습니다.

인플루언서는 앱 진입 즉시 자신의 성과(포인트/페이백)와 활동 상태를 파악할 수 있으며, 복사 기능을 통해 홍보용 코드를 쉽게 가져갈 수 있습니다. Week 4를 통해 인플루언서 가입부터 캠페인 탐색, 지원, 대시보드 확인까지의 핵심 사용자 여정(End-to-End User Journey)이 완성되었습니다.
