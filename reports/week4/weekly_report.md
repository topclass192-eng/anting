# Phase 1 Week 4 회고 및 통합 테스트 시나리오

## 1. Week 4 핵심 달성 사항 (Influencer Core Flow)
이번 Week 4의 핵심 목표는 **인플루언서 사용자의 메인 여정(User Journey)을 구축**하는 것이었습니다. 프로필 등록부터 외부 API 연동, 캠페인 탐색, 지원 및 대시보드 진입까지 End-to-End 플로우가 성공적으로 구현되었습니다.

- **프로필 & 셀러 코드**: 인플루언서 프로필 작성 시 고유 판매자 코드(`sellerCode`, `ANT-XXXXXXXX` 포맷) 자동 발급 로직 완성.
- **인스타그램 연동**: Instagram Basic Display API를 연동하여 OAuth 인증을 통한 계정 연결 기능을 구현하였으며, 에러 발생 시 수동 입력으로 자동 전환되는 폴백(Fallback) 처리 설계.
- **캠페인 탐색**: Firebase Firestore의 `cursor` 및 `startAfter` 기능을 이용해 카테고리, 플랫폼, 지역별 필터링과 무한 스크롤(Infinite Pagination)이 적용된 고성능 리스트 뷰 구축.
- **안전한 캠페인 지원 (트랜잭션)**: 선착순이나 치열한 지원 경쟁 시 발생할 수 있는 동시성 이슈를 막기 위해, 백엔드 Cloud Functions(`apply.ts`)에서 Firestore Transaction 기반 검증 로직 도입 (마감일, 모집초과, 중복신청 검증).
- **대시보드 & 모바일 UI**: 글래스모피즘이 가미된 수익 카드, 원터치 셀러 코드 복사, 권한별 하단 탭(`BottomNav`) 분기를 통해 모바일 앱 라이크(App-like) 사용성 제공.

## 2. 통합 테스트 시나리오 (E2E)

다음은 Week 4 개발분을 기반으로 시스템 전반을 검증하는 통합 테스트 체크리스트입니다.

### ① 인플루언서 로그인 및 프로필 등록 확인
1. 카카오/네이버 모의 로그인 후 역할 선택 화면에서 "인플루언서" 선택.
2. 프로필 입력 폼 진입: 닉네임, 관심사, 지역 등을 입력 후 "저장".
3. **[검증 포인트]**
   - Firestore `influencers` 컬렉션에 유저 문서가 정상 생성되는가?
   - 문서 내 `sellerCode` 필드가 `ANT-`로 시작하는 8자리 영숫자로 발급되었는가?

### ② 캠페인 탐색 및 필터링 동작 확인
1. 하단 `캠페인` 탭으로 이동.
2. 카테고리 탭 (예: 뷰티, 식품) 클릭 및 우측 상단 필터 드롭다운 조작.
3. 스크롤을 하단으로 내려 추가 데이터 로드(무한 스크롤) 작동 여부 확인.
4. **[검증 포인트]**
   - 조건에 맞는 캠페인만 정확하게 노출되는가?
   - `getCampaigns` Cloud Function 로그에 인덱스 초과 에러가 없는가? (필요시 복합 인덱스 생성 확인)

### ③ 중복 신청 방지 및 동시성 트랜잭션 테스트
1. 캠페인 리스트 중 하나를 클릭해 상세 페이지 진입.
2. 하단 "신청하기" 버튼 클릭 → 모달 팝업으로 "신청 완료" 확인.
3. 동일한 기기/계정에서 다시 "신청하기" 버튼 활성화 시도.
4. **[검증 포인트]**
   - 두 번째 신청 시, "이미 신청한 캠페인입니다." Alert 에러 메시지가 정상 노출되는가?
   - Firestore `campaigns` 문서의 `currentApplicants` 필드가 1만 올바르게 증가하였는가?

### ④ 대시보드 상태 및 권한 기반 렌더링 확인
1. 하단 `홈` 탭으로 이동 (인플루언서 대시보드 진입).
2. 자신의 닉네임과 발급된 `sellerCode` 일치 여부 확인.
3. 복사 버튼 클릭 후 클립보드 페이스트 검증.
4. 하단 네비게이션이 브랜드 탭(리포트 등)이 아닌 인플루언서 전용 탭(AI콘텐츠 등)으로 노출되는지 점검.

## 3. 차주(Week 5) 주요 계획 및 연결점
Week 4에서 인플루언서가 캠페인에 지원하는 로직(`applications` 컬렉션 데이터 적재)을 완비했습니다.
이제 **Week 5**에서는 브랜드 사용자가 이를 넘겨받아 **신청자 목록을 조회하고, 최종 인원을 선정(승인/거절)하는 관리자 플로우**를 개발할 예정입니다. 더불어 알림(Notification) 생성 로직을 덧붙여 승인 시 인플루언서에게 피드백이 가도록 파이프라인을 확장합니다.
