# Phase 1 Week 5 Day 21 - 브랜드 신청자 목록 & 선발 UI

## 개요
브랜드 관리자가 자신이 생성한 캠페인에 지원한 인플루언서들의 목록을 조회하고, 다양한 조건으로 필터링하여 원하는 인플루언서를 선발(Selected)하거나 탈락(Rejected)시킬 수 있는 전체 플로우를 구현했습니다.

## 주요 구현 사항

### 1. Backend: Cloud Functions 연동

#### `getApplications`
- **역할**: 캠페인 ID를 받아 해당 캠페인의 지원서 목록(`applications`)과 지원자의 상세 정보(`influencers`)를 병합하여 반환합니다.
- **특징**:
  - `applications` 조회 후 `Promise.all`을 사용하여 병렬로 각 `uid`에 해당하는 `influencers` 문서를 조회했습니다.
  - 조인(Join)된 데이터를 기반으로 백엔드에서 **메모리(서버) 필터링**을 수행합니다. 팔로워 수, 카테고리, 인플루언서 등급(Tier) 필터를 통해 클라이언트에 필요한 데이터만 최적화하여 넘겨줍니다.

#### `selectApplicant`
- **역할**: 브랜드 관리자가 특정 인플루언서를 선발하거나 탈락 처리하는 로직을 담당합니다.
- **특징 (Transaction)**:
  - 동시성 문제가 발생할 수 있으므로, Firestore `runTransaction`을 도입했습니다.
  - '선발(selected)' 시도 시, 캠페인의 `recruitedCount`(선발 완료 인원)를 조회하고 모집 인원(`recruitmentCount`) 초과 여부를 원자적으로 검사합니다.
  - 제한 인원에 도달하면 에러(`resource-exhausted`)를 발생시키며, 정상 처리 시 모집 인원을 +1 하고 애플리케이션의 `status`를 업데이트합니다.

### 2. Frontend: `ApplicantList.tsx`

- **화면 구조**:
  - **헤더 & 통계**: 캠페인명과 모집 현황(현재 선발 인원 / 전체 모집 인원)을 상단에 고정 표출합니다.
  - **필터 바**: 등급, 카테고리, 팔로워 구간(드롭다운 형태)을 제공해 지원자를 손쉽게 검색할 수 있습니다. 횡스크롤(`overflow-x-auto`)을 지원하여 모바일 친화적으로 구성했습니다.
  - **상태 탭**: '대기중(pending)'인 전체 신청자와 '선발 완료(selected)' 탭으로 구분하여 명확한 관리 동선을 제공합니다.
- **신청자 카드 UI**:
  - 인플루언서의 썸네일, 닉네임, 등급, 주요 카테고리 및 SNS 팔로워 합계를 표시하여 선발 기준에 필요한 핵심 요약 정보를 한눈에 볼 수 있도록 설계했습니다.
- **액션 연동**:
  - 🟢선발 / 🔴탈락 버튼 클릭 시 즉시 UI의 옵티미스틱 업데이트(상태 변이)가 일어나 사용자가 지연 없이 체감할 수 있도록 했으며, 하단 "선발 완료" 고정 버튼으로 대시보드 귀환을 돕습니다.
- 라우팅은 `/brand/campaigns/:id/applicants`로 `App.tsx`에 등록되었습니다.

## 기대 효과
- 트랜잭션을 통한 선발 인원 동시성 제어로 정원을 초과해 선발하는 사고를 방지합니다.
- 브랜드는 엑셀 등의 외부 툴 없이, 앱 내에서 모든 지원자 정보를 열람하고 클릭 한 번으로 손쉽게 선발/관리 업무를 마칠 수 있습니다.
