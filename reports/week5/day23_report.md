# Day 23: 제품 수령 확인 & 콘텐츠 제출 (인플루언서) 시스템 구현

## 1. 구현 내용

### 1.1 캠페인 진행 현황 스텝퍼 (`CampaignProgress.tsx`)
- 선발 완료, 배송 중, 제품 수령 확인, 콘텐츠 작성 중, 제출 완료 등 5단계로 캠페인의 생명주기를 시각화하는 세로형 스텝퍼 UI를 도입.
- 브랜드에서 운송장을 입력하면 "배송 중" 단계로 넘어가며, 인플루언서는 "제품 수령 확인" 액션 버튼을 활성화할 수 있음.
- 상태값에 따라 과거, 현재, 미래 단계를 색상(파란색, 회색 등)과 체크마크로 직관적으로 구분.

### 1.2 제품 수령 확인 (`confirmDelivery.ts`)
- 인플루언서가 수령 확인 시 `applications` 데이터의 `contentStatus`를 `writing`으로 업데이트.
- 운송장 번호가 입력되어 있는지(즉, 진짜 배송이 시작되었는지)에 대한 백엔드 방어 로직 추가.
- `deliveryConfirmedAt` 타임스탬프 기록.

### 1.3 콘텐츠 제출 폼 및 수동 검증 (`ContentSubmit.tsx`)
- 인스타그램, 네이버 블로그, 틱톡 등 실제 업로드 플랫폼을 선택하는 UI 구성.
- 콘텐츠 URL 입력란 제공 (입력 즉시 클릭 가능한 링크 렌더링).
- **수동 가이드 체크 로직**: 클라이언트 사이드 웹 스크래핑(OG 태그 파싱 등)의 한계와 CORS 이슈를 고려하여, 인플루언서가 캠페인 필수 가이드(필수 문구, 해시태그, 공정위 문구)를 준수했음을 직접 체크박스로 서약하는 방식으로 우회 구현.
- 3개의 체크박스가 모두 체크되지 않으면 제출 불가 처리.

### 1.4 콘텐츠 제출 처리 (`submitContent.ts`)
- 제출 시 `contentStatus`를 `submitted`로 변경.
- 제출한 `contentUrl`, `platform`, `contentSubmittedAt` 기록 저장.
- 상태 변화 후 다시 `CampaignProgress`로 리다이렉션하여 제출 완료된 스텝퍼를 보여줌.

## 2. 주요 의사결정 및 기술 포인트
- **CORS 및 스크래핑 제약 우회**: 외부 SNS URL 본문을 브라우저 단에서 파싱하는 것은 보안 정책(CORS)으로 불가능하므로, 시스템 복잡도를 낮추고 신뢰성을 확보하기 위해 '인플루언서 자가 진단(수동 체크박스)' 로직으로 타협함. 
- **Application 기반 라우팅**: 캠페인 단계 화면(`/influencer/campaigns/:id/progress`)의 파라미터 `id`를 `campaignId`가 아닌 `applicationId`로 설계하여, 1:1 매핑되는 신청 내역의 상태(`status`, `contentStatus`, `trackingNumber`)를 바로 접근 가능하게 함.
