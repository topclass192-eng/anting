# Day 27: 인플루언서 통합 대시보드 및 마이페이지 완성 보고서

## 1. 개요
- **목적**: 인플루언서(체험단) 사용자가 본인의 수익 및 진행 중인 캠페인을 실시간으로 파악하고, 개인 프로필과 등급, 판매자 코드를 관리할 수 있는 메인 화면(Dashboard) 및 마이페이지(MyPage) 구현
- **주요 파일**: 
  - `apps/web/src/pages/influencer/Dashboard.tsx` (대시보드 화면)
  - `apps/web/src/pages/influencer/MyPage.tsx` (마이페이지 화면)
  - `apps/web/src/App.tsx` (라우팅 추가)

## 2. 주요 구현 내용

### 2.1. 인플루언서 대시보드 (`Dashboard.tsx`)
Firestore의 `onSnapshot`을 통해 실시간 데이터를 동기화하여 인플루언서에게 필요한 핵심 정보를 카드 UI로 제공합니다.

- **수익 요약 카드**:
  - `이번 달 페이백`: 프로필의 `payback` 필드를 연동 (Phase 5 정산 로직 고도화 시 실제 데이터 반영 예정).
  - `출금 가능 포인트`: 보유 중인 `points` 실시간 연동, 클릭 시 포인트/수익 관리 페이지로 즉시 이동.
  - `판매자 코드`: 할당된 고유 `sellerCode` (ANT-XXXX) 표시 및 간편 복사 버튼 연동.

- **마감 임박 알림 배너**:
  - 참여 중인 캠페인 중 "제출되지 않은 상태"이면서 마감 기한이 **3일 이내(D-3 이하)**인 캠페인이 존재할 경우 상단에 붉은색 맥박 애니메이션(`animate-pulse`) 알림 배너 노출.

- **진행 중 캠페인 리스트 (onSnapshot 실시간 연동)**:
  - 현재 사용자의 전체 `applications`를 조회하여 진행 중(`applied` ~ `submitted`)인 데이터 필터링.
  - 캠페인 문서(`campaigns`)를 추가로 조회해 `마감 기한(D-N)`, `제품 썸네일`, `캠페인명` 데이터 병합.
  - **5단계 진행바**: 신청 상태에 따라 `선발 → 배송 → 수령 → 작성 → 제출` 중 현재 단계를 직관적인 프로그레스 바 형태로 시각화.
  - 클릭 시 `캠페인 진행 화면(/influencer/campaigns/:id/progress)`으로 즉시 이동하여 후속 액션 유도.

### 2.2. 마이페이지 (`MyPage.tsx`)
인플루언서의 개인 설정, 프로필 조회, 로그아웃 기능을 담당합니다.

- **프로필 요약 및 수정**: 현재 닉네임과 프로필 이미지를 표시하며, "프로필 수정" 버튼 클릭 시 `/influencer/profile` 양식으로 이동.
- **등급 및 포인트 현황**: 
  - 현재 등급(BRONZE, SILVER 등)과 총 보유 포인트를 강조하여 표시.
  - 다음 등급업을 위해 필요한 잔여 포인트를 계산해 시각적 진행바(`Progress Bar`)로 제공하여 활동 동기 부여.
- **판매자 코드 공유 기능**: `navigator.clipboard`를 통한 복사 기능뿐만 아니라, 모바일 기기를 고려한 네이티브 공유 API(`navigator.share`) 기능 연동.
- **기타 메뉴 및 로그아웃**: 포인트 내역 이동, 고객센터, 이용약관 메뉴를 구현하고 최하단에 안전한 Firebase 로그아웃 처리(`signOut`) 구현.

## 3. 결과 및 향후 계획
- 인플루언서는 본인의 캠페인 진행 상황을 단계별로 명확히 인지할 수 있게 되었으며, 마감일 임박 알림을 통해 콘텐츠 지연 제출을 효과적으로 방지할 수 있는 UI 구조가 확립되었습니다.
- 대시보드 내 "이번 달 페이백" 등 일부 수익 기능은 향후 Phase 5의 결제/정산 연동 파트에서 백엔드 로직이 완성되는 대로 최종적으로 맵핑할 예정입니다.
